Sessions

All API calls require authentication using a session key. Session keys are unique strings. You can access any auth-only page on http://fwol.in/ by adding a ?sessionid< parameter. So http://fwol.in/api/me?sessionid=a-uuid is an easy check to get a user's profile information and see if they are still logged in.

There are two ways to get a session ID from fwol.in:

  1. Client: POST https://ohack-fwolin.herokuapp.com/api/exchangelogin with the form values username and password. This returns 401 if invalid, or 200 with a JSON body {"sessionid": "a-uuid"}.
  2. Browser: Send the browser to http://fwol.in/login/?callback=http://map.olinapps.com/login/&external Note the "external" key. If the callback URL is valid (olinapps, fwol.in, or localhost) it redirects the browser to the callback URL with a "sessionid" parameter, i.e. http://map.olinapps.com/login/?sessionid=a-uuid The app should save this key and consider the user logged in. It can check http://fwol.in/api/me to see if the credentials are still valid.

There will be a Python module for automating this soon.
